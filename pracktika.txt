**Тема:** Програмний засіб для автоматизації роботи пункту прокату спортивного спорядження
UI Framework:**  AvaloniaUI (xaml)
*Файлове сховище:** JSON (Data/users.json, Data/equipment.json, Data/customers.json, Data/rentals.json
**Зображення:** /images/equipment/{id}.jpg
- **Min .NET:** 8.0
**Мова:** C#
1. Сутності (min 4, можна більше)

| Entity | Fields |
|--------|--------|
| **User** | id, login, passwordHash, role (Admin/Operator), fullName, createdAt |
| **Equipment** | id, name, category, dailyRate, depositAmount, quantity, condition (Good/Damaged/UnderRepair), imagePath, createdAt |
| **Customer** | id, fullName, phone, passportNumber, createdAt |
| **Rental** | id, customerId, userId, rentalDate, dueDate, returnDate, status (Active/Returned/Overdue), totalAmount, depositPaid |
| **RentalItem** | id, rentalId, equipmentId, quantity, dailyRate, days, subtotal |

 2. Зв'язки
    User ||--o{ Rental : "оформлює"
    Customer ||--o{ Rental : "орендує"
    Rental ||--o{ RentalItem : "включає"
    Equipment ||--o{ RentalItem : "позичається"
   

3. Функціональність

### Аутентифікація та авторизація
- Вхід працівника (логін + пароль)
- Роль Адмін: повний доступ + звіти 
- Роль Оператор: оформлення оренди, повернення
- Обов'язкове хешування паролів (BCrypt, Argon2 або PBKDF2)

### Валідація
- Валідація даних клієнта (паспорт, телефон)
- Перевірка наявності обладнання
- Валідація дат (dueDate > rentalDate)

### CRUD операції
| Entity | Create | Read | Update | Delete |
|--------|--------|------|--------|--------|
| User | + | + | + | + |
| Equipment | + | + | + | + |
| Customer | + | + | + | + |
| Rental | + | + | + | + |
| RentalItem | + | + | + | + |

### Пошук, фільтрація, сортування, пагінація
- **Пошук:** за назвою обладнання, ПІБ клієнта, номером оренди
- **Фільтрація:** за статусом оренди, категорією обладнання, датою
- **Сортування:** за датою, сумою, назвою (asc/desc)
- **Пагінація:** для списку оренд та обладнання (20/50/100 на сторінку)

### Бізнес-логіка
1. **Оренда:**
   - Вибір обладнання + клієнт
   - Вказання терміну оренди
   - Автоматичний розрахунок суми (dailyRate * days)
   - Внесення застави (deposit)
   - Бронювання обладнання

2. **Повернення:**
   - Фіксація повернення
   - Перевірка стану обладнання
   
3. **Система штрафів:**
   - Налаштування штрафу за день прострочення
   - Автоматичне нарахування при поверненні
   - Можливість ручного нарахування

4. **Обладнання:**
   - Кількість одиниць
   - Стан: Добре / Пошкоджено / На ремонті

## 4. Сценарії використання (Flow)

### Flow 1: Оформлення оренди
```
1. Оператор -> "Нова оренда"
2. Вибрати клієнта (новий або існуючий)
3. Додати обладнання (з вибором кількості)
4. Вказати термін (від -> до)
5. Автоматичний розрахунок суми
6. Внесення застави
7. Підтвердження -> оренда активна
8. Зменшення available quantity обладнання
```

### Flow 2: Повернення обладнання
```
1. Оператор -> "Повернення"
2. Знайти оренду (по клієнту або номеру)
3. Перевірити стан обладнання
4. Оновлення статусу оренди
5. Повернення quantity обладнання
